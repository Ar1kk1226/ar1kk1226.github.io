# Git的使用

## 1. 初始化与配置

在开始之前，需要确保 Git 已安装，并配置用户身份，可以进行全局配置（只需一次），也可以本地配置（适合多账户公用机器）

- 配置用户信息

配置用户信息‌（全局配置，只需设置一次）：

     git config --global user.name "你的名字"

     git config --global user.email "你的邮箱@example.com"

或 本地配置：
     git config --local user.name "你的名字"
    
     git config --local user.email "你的邮箱@example.com"

- 初始化本地仓库‌：

进入项目根目录，执行以下命令创建一个新的 Git 仓库。这会在目录下生成一个隐藏的 .git文件夹，用于存储版本控制信息。

   git init


## 2. 添加文件到暂存区 (Stage)

Git 的工作流程中，修改后的文件首先处于“未跟踪”或“已修改”状态，需要将其添加到“暂存区”才能准备提交。

- 添加指定文件‌：
	git add 指定文件

- 添加所有变更文件‌：

    git add .

## 3. 提交到本地仓库 (Commit)

将暂存区的文件快照永久保存到本地版本历史中。每次提交都应附带清晰的说明信息。

执行提交‌：
    
    git commit -m "本次提交的简要说明，例如：修复登录bug"

其中，-m 参数后跟随的是提交日志消息，简明扼要地描述本次修改的内容即可。

## 4. 查看状态与历史

在操作过程中，随时查看文件状态和提交记录有助于管理版本。

查看当前状态‌，显示哪些文件被修改、哪些在暂存区、哪些未跟踪。
    
     git status

查看修改详情‌，比较工作区文件与暂存区或最新提交的差异：

    git diff       # 查看工作区与暂存区的差异
    
    git diff HEAD  # 查看工作区与最新提交的差异
