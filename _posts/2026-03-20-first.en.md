## 1. Initial & Config

Before starting, it is necessary to ensure that Git is installed and configured with user identity, which can be configured globally (only once) or locally (suitable for multi account shared machines)

- Configure user information

Configure user information (global configuration, only needs to be set once)

     git config --global user.name "yourname"

     git config --global user.email "youremail@example.com"

or local configure ：
     
     git config --local user.name "yourname"
    
     git config --local user.email "youremail@example.com"

- Initialize local repository:

Enter the root directory of the project and execute the following command to create a new Git repository. 
This will generate a hidden .git folder in the directory for storing version control information,and DON'T DELETE THE FOLDER

     git init


## 2. Add files to the staging area (Stage)

In the Git workflow, the modified file is first in the "untracked" or "modified" state and needs to be added to the "Stage area" before it can be ready for submission.

- Add specified file:

	  git add file

- Add all files

      git add .

## 3. Submit to local repository (commit)

Permanently save the file snapshot of the temporary storage area to the local version history. Each submission should be accompanied by clear explanatory information

执行提交‌：
    
    git commit -m "A brief description of this commit, such as fixing login bugs"

-m  parameter is followed by a submission log message, which briefly describes the content of this modification.

## 4. View status and history

Checking file status and submission records at any time during the operation process helps manage versions.

View the current status and display which files have been modified, or in the staging area, or not tracked.
   
     git status

View the modification details and compare the differences between the workspace file and the temporary storage or the latest submission:

    git diff       # view differences between workspace and stage area
    
    git diff HEAD  # View differences between workspace and latest commit

View commit log:
    
    git log

## 5, Revocation and version rollback

Revoke workspace modifications before adding:
    
    git checkout filename

When adding but not committing, undo the modifications:

    git reset HEAD filename          #（low git verision）
    git restore --staged filename    # （new version）

Rollback to Previous Versions 

    git reset --hard HEAD~1      # Rollback to the previous version, discard all modifications
    git reset --soft HEAD~1      # Rollback to the previous version and keep the changes in the temporary storage area

## 6. Submit remotely or obtain remotely

Submit to remote server
     
     git push

Retrieve from remote server

    git clone      # Clone remote repository
    git fetch      # Synchronize updates from remote to local
